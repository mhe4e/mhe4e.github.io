    Website

    We appreciate everyone's alerts about the events section of the website, as well as other concerns. As you can imagine, there is some complex coding for the site to work seamlessly with SalesForce and we have had to deal with a few glitches. All urgent matters should be sent to digital@e4e.org.

    What's been fixed:
    • • Events are now filtered to only show those marked “Display on Website” in Salesforce.
    • • All events are now registering users for the correct event. (There were a few events that were logging registrations for other events.)
    • • For events that do not require membership to attend, non-members are now passed to a shorter page (viewable here) instead of being directed to the Declaration.
    • • Events are now showing the correct start and end dates and times.
    • • The correct event links are showing up in Salesforce; these are viewable in the External Event URL field (the same place where links showed up before).
    • • We developed a process to ensure the most recent content (ie the week's TTB) will be showcased.
    What's in the works:
    • • For some users, the Declaration is not directing them to the thank you page after they click “Sign the Declaration." This is a high priority item, but the good news is the website is accurately logging online signups and no data is being lost.
    • • On some events, the “Add to Calendar” button is not appearing.
    • • A number of format corrections are being made throughout the site.
    • • A few site enhancements are also being made from now through the end of March. We are still finalizing this list.
    How you can help with event content:
    You’ll want to pay attention to these three fields when creating events in Salesforce:
    • 1. Display on Website: Check this field if the event should be shown on the main list of events or your chapter’s list of events. If left uncheck, you’ll still be able to share a link, but it won’t appear on the website.
    • 2. Event List Description: Use this to provide the brief event description is listed on the website. (Think 140 characters or less, and avoid repeating the event name.)
    • 3. Description: Here’s where you’ll write the summary of the event that is displayed in detail on the website, otherwise the page is blank.
